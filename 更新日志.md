# 更新日志 (Changelog)

## [v2.0.1] - 2026-03-28

### 🚀 新功能与部署优化
- **Docker 容器化支持**：
  - 新增 `docker/Dockerfile` 与 `docker-compose.yml`，支持一键部署。
  - 默认服务端口统一调整为 `9090`。
  - 配置了持久化数据卷（`/app/data` 和 `/app/logs`），确保数据库与日志在容器重启后不丢失。
- **GitHub 仓库同步**：
  - 全量项目已推送至 GitHub 仓库：`https://github.com/jeja2023/CPACM.git`。
  - 完善了 `README.md` 的中文快速部署说明。

### 🔧 核心组件优化 (性能与稳定性)
- **Cloudflare 403 绕过方案升级**：
  - 优化了 `src/core/http_client.py` 中的浏览器指纹模拟，升级至 `chrome110` 版本。
  - 移除了硬编码的 `User-Agent`，改由 `curl_cffi` 内部自动生成，彻底解决因指纹不一致导致的 403 拦截问题。
- **数据库查询性能提升**：
  - 在 `Account` 模型的 `status`、`email_service`、`cpa_uploaded` 关键字段上增加了数据库索引（Indexes），大幅加快海量账号下的筛选与导出速度。
- **自动内存回收机制**：
  - 在 `TaskManager` 中引入了异步定时清理任务，每小时自动清除超过 24 小时的已完成/失败任务的历史日志与中间状态，防止系统长时间运行导致内存膨胀 (OOM)。
- **流式导出处理 (Anti-OOM)**：
  - 重构了账号管理中的 JSON/CSV/Sub2API 导出接口，全面采用 `StreamingResponse` 流式输出。数据边查边推，即便导出上万个账号也不会瞬间撑爆服务器内存。

### 📝 其他
- 全面优化了核心逻辑中的中文代码注释。
- 标准化了项目根目录结构。
