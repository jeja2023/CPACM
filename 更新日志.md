# 更新日志 (Changelog)

## [v2.0.2] - 2026-03-28

### 🛠️ CPA 功能与 UI 深度优化 (v2.0.2)
- **数据一致性同步**：修复了“账号管理”与“CPA 检测”页面统计不一致的问题。现在从 CPA 平台删除/清理账号时，本地数据库会自动同步将 `cpa_uploaded` 标记为 `False`，并根据状态转入“失效”或“失败”类别。
- **检测页面统计回显**：优化了 CPA 检测页面初次加载逻辑。现在页面会自动拉取并回显最近一次后台巡检的统计结果（401、额度耗尽、异常数）到顶部卡片，避免页面打开时显示全 0 的空数据。
- **新增手动同步功能**：在 CPA 运行控制台增加了“同步账号”按钮，支持手动强制刷新账号列表和巡检统计，提升操作实时性。
- **邮箱解析算法增强**：改进了 `perform_scan` 中的账号提取逻辑，支持从 `.json` 后缀的文件名中智能恢复邮箱，确保封号状态（Banned/Expired）能精准同步回本地数据库记录。

### 🐛 严重缺陷修复
- **邮箱服务管理按钮修复**：彻底修复了“邮箱服务管理”页面点击“添加服务商”无反应的问题。补全了模板中丢失的 `add-custom-modal` 各子类型表单字段及 `edit-modal` 编辑模态框 HTML，消除了 JS 初始化时的 `TypeError` 报错。

### 📡 网络与稳定性增强
- **请求超时与容错**：在 `list_accounts` 接口中增加了针对 `requests.exceptions.ReadTimeout` 和 `ConnectionError` 的专门捕获。
- **超时策略调整**：将 CPA 接口请求超时从 15s 提升至 **20s**，并增加了友好的中文错误反馈提示（如“连接 CPA 服务器超时”），极大提升了网络波动环境下的用户体验。


## [v2.0.1] - 2026-03-28

### 🛠️ 性能与稳定性优化 (v2.0.1)
- **数据库索引优化**：为 `Account` 模型中常用的过滤字段（状态、服务类型、上传标识）增加索引，大幅提升海量数据下的查询加载速度。
- **内存回收机制**：在 `TaskManager` 中引入自动内存扫描与回收。定期清理过期的任务日志、锁及缓存，彻底避免长运行下的内存膨胀。
- **流式数据导出**：重构 `JSON/CSV` 及 `Sub2Api` 导出接口，采用 Python 生成器（Streaming）配合 SQLAlchemy `yield_per` 进行低内存迭代处理，彻底杜绝大批量数据导出导致的 OOM (Out-of-Memory) 崩溃。

### 🐛 缺陷修复
- **巡检启动异常修复**：修正了 `AutoPatrolManager` 在应用模块加载时抢先解析异步配置导致的 `no running event loop` 运行时错误，及协程未等待的警告。
- **统计数据校准**：完善了 CPA 巡检（cliproxy）的结果反馈链路。检测到 401（Token 无效）或额度耗尽后，会自动同步更新本地 `accounts` 表状态，确保账号管理页面的统计卡片始终保持准确。
- **CF 403 验证修复**：修复了在特定环境下 Cloudflare 403 拦截导致的登录/刷新失败问题，增强了请求头的伪装与通过率。
GitHub 仓库：`https://github.com/jeja2023/CPACM.git`。
  - 完善了 `README.md` 的中文快速部署说明。

### 🔧 核心组件优化 (性能与稳定性)
- **Cloudflare 403 绕过方案升级**：
  - 优化了 `src/core/http_client.py` 中的浏览器指纹模拟，升级至 `chrome110` 版本。
  - 移除了硬编码的 `User-Agent`，改由 `curl_cffi` 内部自动生成，彻底解决因指纹不一致导致的 403 拦截问题。
- **数据库查询性能提升**：
  - 在 `Account` 模型的 `status`、`email_service`、`cpa_uploaded` 关键字段上增加了数据库索引（Indexes），大幅加快海量账号下的筛选与导出速度。
- **自动内存回收机制**：
  - 在 `TaskManager` 中引入了异步定时清理任务，每小时自动清除超过 24 小时的已完成/失败任务的历史日志与中间状态，防止系统长时间运行导致内存膨胀 (OOM)。
- **流式导出处理 (Anti-OOM)**：
  - 重构了账号管理中的 JSON/CSV/Sub2API 导出接口，全面采用 `StreamingResponse` 流式输出。数据边查边推，即便导出上万个账号也不会瞬间撑爆服务器内存。

### 🐛 Bug 修复
- **自动巡检初始化修复**：解决了 CPA 自动巡检模块在应用刚启动时由于 `asyncio` 事件循环尚未就绪而尝试创建异步任务导致的 `no running event loop` 报错。

- 全面优化了核心逻辑中的中文代码注释。
- 标准化了项目根目录结构。
